Небольшое fullstack-приложение с авторизацией по JWT, CRUD для пользователей и постов, PostgreSQL и Redis-кэшем.

- FastAPI

- Redis
- JWT

### Frontend
- Vue 3
- Vite
- Vanilla CSS

- PostgreSQL 15
- Redis 7

## Как пользоваться

### 1. Создайте пользователя
На главной странице введите имя и нажмите **Создать и войти**.

## Основные endpoints

### Пользователи

#### Создать пользователя
```http
POST /users
```

Body:

  "name": "Иван"
}
```

Ответ:

  "access_token": "jwt_token",
  "token_type": "bearer"
}
```

---

#### Получить токен по ID пользователя
```http

```

---

#### Получить текущего пользователя
```http
GET /users/me
```

---

#### Обновить текущего пользователя
```http
PATCH /users/me
```

Body:
```json
{
  "name": "Новое имя"
}
```

---

#### Удалить текущего пользователя
```http

```

---

### Посты

#### Создать пост
```http
POST /posts
```

Body:
```json
{
  "title": "Заголовок",
  "text": "Текст поста"
}
```

---

#### Получить пост по ID
```http
GET /posts/{post_id}
```

---

#### Обновить пост
```http
PATCH /posts/{post_id}
```

Body:
```json
{
  "title": "Новый заголовок",
  "text": "Новый текст"
}
```

Можно передавать только одно поле.

---

#### Удалить пост
```http
DELETE /posts/{post_id}
```

---

#### Получить свои посты
```http
GET /users/me/posts?limit=10&offset=0
```

Параметры:
- `limit` — количество записей
- `offset` — смещение

## Кэширование

Посты кэшируются в Redis, если они считаются “горячими” — то есть созданными или обновлёнными менее чем 10 минут назад.

Логика:
- при создании поста он сразу кладётся в кэш
- при чтении сначала проверяется Redis
- если кэш устарел, данные берутся из PostgreSQL
- после обновления поста кэш инвалидируется и при необходимости обновляется

## Возможные проблемы

### 1. Проект не запускается из-за занятого порта
Если порты `5173`, `8000`, `5432` или `6379` уже заняты, остановите другие сервисы или поменяйте порты в `docker-compose.yml`.

### 2. База данных не поднимается
Иногда при первом запуске PostgreSQL инициализируется дольше обычного.  
Подождите немного и попробуйте снова.

### 3. Токен потерян
Если токен удалён из localStorage или истёк, просто создайте пользователя заново или получите токен по ID.
